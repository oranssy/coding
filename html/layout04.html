레이아웃04

레이아웃(layout)이란, 각 구성요소를 특정한 공간 안에 효과적으로 배열하는 일을 말합니다.
활동 또는 앱 위젯의 UI와 같은 사용자 인터페이스의 시각적 구조를 정의합니다.
   가운데 영역에서 5칸으로 나뉜 레이아웃을 만들어봅니다.

01. float 활용하기

float 의 원래 기능은 글씨와 이미지를 정렬하는 것입니다.
이전에 레이아웃을 설정할 방법이 없을 때 사용하였으며, 설정한 칸을 왼쪽 또는 오른쪽 위로 띄워서 빈 공간을 채워줍니다.
    ** 참고하기: 틀을 구성하는 div 박스는 block 구조를 기본 성질로 가지며, 자동으로 세로 정렬이 됩니다.

float 설정 이후에 aside칸과 section칸의 'height' 값을 0으로, 즉 빈 칸으로 인식하는 버그가 발생합니다.
이에 따라 footer칸이 빈 칸을 채우기 위해 위로 올라가 숨는 모양이 되므로, 이를 해결하기 위해 clear: both; 를 넣어줍니다.
    ** 참고하기: clear: both; 는 빈 공간으로 인식한 칸이 채워져 있다고 알려주는 역할을 합니다.

02. flex 활용하기

flex는 레이아웃 배치 전용 기능으로 고안되었습니다.
부모-자식 관계에서 유지되는 원리이며, 이는 전체 공간 안에서 설정된 속성에 따라 구성요소가 배치되는 것을 의미합니다.

flex 를 활용하기 위해 제일 먼저 기본 태그를 시멘틱 태그로 바꿔줍니다.
    **참고하기: span, div 태그는 의미를 갖지 않으므로 이름을 지어 의미를 부여할 수 있습니다.
기본이 block 구조이므로, 부모 요소인 #wrap 에 display: flex; 를 넣어서 가로 정렬로 바꿔줍니다.
자식 요소의 치수를 기준으로 줄을 넘기기 위해 flex-wrap:wrap; 을 넣어줍니다.
    ** 참고하기: float 보다 훨씬 간단하지만 IE 버전에서는 적용이 되지 않으며, 이런 경우는 float를 씁니다.

03. grid 활용하기

grid 는 flex 가 발전한 형태로, flex 보다 복합적인 레이아웃 표현이 가능합니다.
    ** 참고하기: flex 는 가로와 세로 중에 한 방향으로만 설정할 수 있고, grid 는 가로와 세로의 두 방향으로 설정이 가능합니다.
flex 와 마찬가지로 부모-자식 관계에서 유지되는 원리를 가집니다.

부모 요소인 #wrap 에 display: grid; 를 넣어 중복되는 값은 부모를 따라가도록 합니다.
grid-template-areas; 를 넣고 테이블이 차지하는 공간을 세분화하여 이름을 지정해줍니다.

grid-template-columns; 를 넣어서 가로 '열'의 비율을 설정합니다.
grid-template-rows; 를 넣어서 세로 '행'의 비율을 설정합니다.
